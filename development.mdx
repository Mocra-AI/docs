<Info>
  **Prerequisites:** Node.js version 19 or higher, and a clone of this docs repository.
</Info>

This guide is for anyone editing the Mocra documentation. The site is built with [Mintlify](https://mintlify.com). Use the Mintlify CLI to preview changes locally before pushing.

<Steps>
<Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```

</Step>

<Step title="Preview the docs locally">

From the root of this repo (where `docs.json` lives), run:

```bash
mint dev
```

Open `http://localhost:3000` to view the Mocra docs. Edits to MDX and OpenAPI files will hot-reload.

</Step>
</Steps>

## Custom port

Mintlify defaults to port 3000. To use another port (e.g. 3333):

```bash
mint dev --port 3333
```

If the chosen port is in use, the CLI will try the next available port.

## Keeping the CLI up to date

Each Mintlify CLI release is tied to a specific docs runtime. If your local preview looks different from production, update the CLI:

```bash
mint update
```

## Checking links

To find broken links in the documentation:

```bash
mint broken-links
```

## Deploying changes

Pushes to the default branch are deployed automatically via the Mintlify GitHub integration. A successful deployment shows "All checks have passed" in the repo.

## Editing tips

- **MDX:** For syntax highlighting and formatting in VS Code, use the [MDX extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) and [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode).
- **API reference:** Endpoints and request/response schemas are defined in `api-reference/openapi.yml`. The playground and code samples are generated from that file.

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    This often means Node is too old. Try:
    1. Uninstall the CLI: `npm remove -g mint`
    2. Upgrade to Node v19 or higher.
    3. Reinstall: `npm i -g mint`

  </Accordion>

  <Accordion title="Unknown or odd errors when running mint dev">

    Clear the Mintlify cache: delete the `~/.mintlify` folder from your home directory, then run `mint dev` again.
  </Accordion>
</AccordionGroup>

For Mintlify CLI changes, see the [CLI changelog](https://www.npmjs.com/package/mintlify?activeTab=versions).
